18649 - Distributed Embedded Systems

Distributed Elevator Control System

Project Portfolio

18649 Fall 2014
Group 4 - Zhu Zhang (zhuzhang), Hua Liu (hual1), Howard Lee (hweekeul)

Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - Scenarios describe the way in which user and system interact in each use case and the sequence diagram describes interaction and communication among different components in each scenario.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - A list of system constraints, sensors and actuators requirements, each has assumptions, input, output interfaces, state, constraints and event-triggered behaviors.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - An excel file used for tracing sequence diagrams to requirements.
    • Requirements to Contstraints Traceability - A table used for tracing requirements to constraints.
    • Statecharts to Code Traceability - Document used for trace state chart transitions to code.
  • Implementation
    • Elevator Control Package - A folder contains all 7 controllers - CarButtonControl, CarPositionControl, DoorControl, Dispatcher, DriveControl, HallButtonControl, LanternControl and other classes.
  • Test
    • Unit Test Log - Records of unit test results.
    • Unit Test Summary File - Parseable list of unit test files
    • Integration Test Log - Records of integration test results.
    • Integration Test Summary File - Parseable list of integration test files
    • Acceptance Test Log - Records of acceptance test results.
    • Fault Tolerance Test Log - Records of fault tolerance test results.
  • Log Files
    • Issue Log - Log file for recording found issues of the system.
    • Improvements Log - Advices for improvements of projects.
    • Peer Review Log - Peer review results records.
  • Scheduling
    • Network Schedule - Message dictionary and network schedule
  • Fault Tolerance
    • FMEA - Failure Modes and Effects Analysis
    • FTA - Fault Tree Analysis
  • Verification
    • High Level Requirements Verification Framework - Runtime monitor designs and high level requirements verification tests
  • Presentations
    • 04_mid_pres.ppt - Midterm presentation slides
    • 04_final_pres.ppt - Final presentation slides
